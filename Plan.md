# Plan: Bowling Game Kata — Step 2

## 목표 (이번 단계에서 구현할 동작)

스페어/스트라이크가 전혀 없는, 모든 굴림에서 1핀만 쓰러뜨리는 게임의 점수는 **20점**이다.

(20번 굴림 × 1핀 = 20점. 보너스 로직 없이 단순 합산이 맞는지 한 번 더 확인하는 케이스이며, 다음 단계(스페어/스트라이크)로 넘어가기 전 기존 `score()` 구현이 우연히 거터 케이스에만 맞아떨어진 게 아님을 검증한다.)

## 무엇을 테스트할 것인가

- `test_game.py`에 `test_all_ones_scores_twenty` 테스트 추가
- `Game` 인스턴스를 만들고, 20번 `roll(1)`을 호출한 뒤 `score()`가 `20`을 반환하는지 확인

```python
def test_all_ones_scores_twenty():
    game = Game()
    for _ in range(20):
        game.roll(1)
    assert game.score() == 20
```

## 왜 이 테스트부터인가

- 현재 `score()`는 `sum(self.rolls)`로 이미 이 케이스를 통과할 가능성이 높다 — 즉 RED 검증 시 실패하지 않을 수 있다는 리스크가 있다.
- 만약 이 테스트가 구현 변경 없이 곧바로 통과(green)한다면, 이는 "이미 존재하는 동작을 검증하는 테스트"이므로 새로운 프로덕션 코드 변경 없이 테스트만 추가하고 GREEN 단계를 건너뛰는 대신, 다음 실제 RED를 만드는 케이스(스페어)로 바로 넘어가는 것을 사람 파트너와 논의한다.
- 정공법으로는 다음 케이스가 스페어(spare) 처리이지만, 그 전에 "보너스 없는 합산"이 여러 굴림 수에도 안정적으로 동작함을 짧게 짚고 넘어가고자 한다.

## 어떻게 구현할 것인가 (GREEN 단계 예상 — 최소 구현)

- 별도 구현 변경이 필요 없을 가능성이 높음 (`sum(self.rolls)`가 이미 처리)
- 만약 테스트가 RED로 실패하지 않는다면, 이 사실을 사람 파트너에게 보고하고 이 테스트를 스킵할지, 혹은 바로 다음 목표(스페어 보너스)로 계획을 수정할지 확인받는다.
